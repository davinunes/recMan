Visão Geral

Esta API REST permite integrar e atualizar o sistema de recursos e controle de notificações (ciencia, dados cadastrais e multas). A API é estruturada de forma simples e direta, operando sobre JSON.

Nota sobre Autenticação: A validação de tokens está desabilitada temporariamente para fins de homologação e testes locais. Futuramente, será cobrado o cabeçalho padrão Authorization: Bearer <TOKEN>.

O endpoint base das chamadas locais está localizado em:

/api/recursos.php

GET Listagem de Recursos

Retorna os dados cadastrais dos Recursos criados. Por padrão, retorna todos os registros. É possível filtrar por uma lista específica de números, ou por mês e ano.

Adicionalmente, se o recurso possuir um Parecer concluído associado, será retornado um objeto parecer simplificado contendo a data, o resultado final, a cópia do PDF gerado em Base64 e o hash SHA-256 deste PDF.

Campos de Retorno (JSON)

Cada item da lista contém dados cadastrais básicos e os novos campos relacionados a retirada e cobrança:

  • id, unidade, bloco, numero, artigo, fase, email, Nome, detalhes, titulo, data, fato.
  • data_retirada: Data em que a notificação foi retirada/ciente (caso cadastrada).
  • observacao_retirada: Observações cadastradas na retirada.
  • data_cobranca: Data de cobrança/vencimento da multa aplicada.
  • valor_cobranca: Valor da cobrança/multa (decimal).
  • parecer: Objeto parecer (se existente).

POST Atualizar Notificação

Permite alterar a data de ciência de notificação e atualizar dados adicionais no cadastro da notificação correlata. O recurso é utilizado apenas como identificador do registro.

A API mapeia e atualiza automaticamente os registros em:

  • DatasDeRetirada: Atualiza a data de ciência (dia_retirada) e observações de entrega.
  • notificacoes: Atualiza dados cadastrais, tais como status, assunto, tipo, valores e datas de envio da notificação.
  • multas_cobradas: Atualiza valores de multa, data de vencimento e pagamento.

Corpo da Requisição (JSON ou Form-Data)

Campo Tipo Obrigatório Descrição
recurso_numero string Parcial* Número do recurso identificador (Ex: 186/2023)
recurso_id int Parcial* ID numérico interno do recurso no banco de dados
data_ciencia string Não Data de ciência da notificação (Ex: 2026-05-29 ou 29/05/2026)
obs_retirada string Não Observações adicionais de retirada física/digital
data_cobranca string Não Ajusta a data de cobrança/vencimento da multa (Ex: 10/06/2026)
valor_cobranca decimal Não Ajusta o valor da cobrança da multa aplicada (Ex: 250.00)
assunto string Não Assunto / Infração vinculada

* É obrigatório o envio de recurso_id OU recurso_numero para localizar o registro.
